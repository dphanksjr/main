Air Hockey Demo


This is a "simple" air hockey demo that I made. To develop this demo I used a three stage software development guide that is broken down as follows:

  • Analysis: This is the studying of the software requirement in detail. This is where a clear pricture of how the program ought to run is realized and what problems will need to be solved in regards to topics such as AI behavior and physics.
  • Design: This stage requires problems/features identified in the first stage to be solved and decisions on how the actual program should be constructed to be made. For this demo algorithmic solutions needed to be created to determine how the game would behave with things such as collisions (puck/paddle, puck/wall, etc.) and how the computer behaves (AI). The slides shown give examples of how the AI was created as well as an example of the collision physics.
  • Implementation: This is the actual creation of the software. This is where the problems identified are solved (and sometimes new problems created). In short, the real work is done here and you find out how well you did in the Design phase.
